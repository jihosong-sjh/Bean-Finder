문서 버전: v0.1

제품명: Bean Finder, 가칭  
연관 문서: [PRD.md](./PRD.md), [FUNCTIONAL_SPEC.md](./FUNCTIONAL_SPEC.md), [SCREEN_SPEC.md](./SCREEN_SPEC.md), [DATA_SPEC.md](./DATA_SPEC.md), [API_SPEC.md](./API_SPEC.md), [TECHNICAL_DESIGN.md](./TECHNICAL_DESIGN.md)  
문서 목적: Bean Finder MVP가 기능, 데이터, API, 화면, 접근성, 성능 기준을 만족하는지 검증하기 위한 테스트 범위와 케이스를 정의한다.

## 1. 범위

이 문서는 MVP 출시 전 검증해야 하는 테스트를 정의한다.

포함 범위:

- 데이터 검증
- 가격 정규화
- 쉬운 맛 표현 매핑
- 검색
- 필터
- 정렬
- 원두 상세
- 원두 비교
- 카테고리
- 랭킹
- 판매처 이동
- 이벤트 로깅
- API 응답
- 화면 QA
- 접근성
- 반응형
- 기본 성능

제외 범위:

- 회원가입과 로그인
- 결제
- 사용자 리뷰 작성
- 찜 목록 서버 저장
- 개인화 추천
- 자동 크롤링
- 관리자 웹 화면
- 모바일 네이티브 앱

## 2. 테스트 전략

### 2.1 테스트 레벨

| 레벨 | 목적 | 대상 |
| --- | --- | --- |
| 단위 테스트 | 순수 로직이 정확히 동작하는지 검증 | 가격 계산, 필터, 정렬, 점수 계산 |
| 데이터 검증 테스트 | 원두 데이터가 명세를 만족하는지 검증 | JSON/CSV/seed 데이터 |
| API 테스트 | 요청/응답 형식과 오류 처리를 검증 | `/api/v1/*` |
| 통합 테스트 | 화면과 API/service 연결을 검증 | 검색 결과, 상세, 비교 |
| E2E 테스트 | 사용자 흐름 전체를 검증 | 검색, 비교, 판매처 이동 |
| 수동 QA | 시각적 품질과 사용성을 검증 | 화면, 반응형, 접근성 |
| 성능 테스트 | MVP 성능 기준을 만족하는지 검증 | 검색, 상세, batch API |

### 2.2 권장 도구

실제 프레임워크 확정 전 권장 도구는 다음과 같다.

| 목적 | 권장 도구 |
| --- | --- |
| 단위 테스트 | Vitest 또는 Jest |
| API 테스트 | Vitest/Jest, supertest 또는 프레임워크 테스트 도구 |
| E2E 테스트 | Playwright |
| 접근성 검사 | axe, Playwright accessibility checks |

| 데이터 검증 | Zod, Valibot, JSON Schema 중 선택 |
| 성능 확인 | Lighthouse, Playwright timing, 서버 로그 |

## 3. 테스트 환경

| 환경 | 목적 | 데이터 |
| --- | --- | --- |
| local | 개발 중 빠른 검증 | 샘플 데이터 30-50개 |
| staging | 출시 전 QA | 실제에 가까운 데이터 300개 이상 |
| production | 출시 후 모니터링 | 운영 데이터 |

MVP 출시 전 최소 조건:

- 샘플 원두 30개 이상
- 로스터리 5개 이상
- 산미 낮음/높음, 바디감 낮음/높음 원두 각각 포함
- 라떼용, 핸드드립용, 디카페인 원두 포함
- 가격대별 원두 포함
- 품절 원두 1개 이상 포함

## 4. 테스트 데이터 기준

테스트 데이터에는 다음 케이스가 반드시 포함되어야 한다.

| 데이터 케이스 | 목적 |
| --- | --- |
| 산미 1-2 원두 | 신맛 적은 검색/카테고리 검증 |
| 산미 4-5 원두 | 산미 있는 검색/랭킹 검증 |
| 바디감 4-5 원두 | 묵직함/라떼용 검증 |
| 2만원 이하 원두 | 가격 필터 검증 |
| 100g당 가격이 낮은 원두 | 가성비/랭킹 검증 |
| 디카페인 원두 | 디카페인 필터 검증 |
| 품절 원두 | 판매 상태 검증 |
| 이미지 없는 원두 | 기본 이미지 fallback 검증 |
| 평점 없는 원두 | 추천순 fallback 검증 |
| 선택 필드 일부 누락 원두 | 상세 화면 `정보 없음` 처리 검증 |

## 9. E2E 테스트

### 9.1 신맛 적은 원두 탐색

Given 사용자가 홈 화면에 있다.  
When `신맛 적은 원두` 카테고리를 선택한다.  
Then 산미 점수 1-2인 원두 목록이 표시된다.  
And 각 카드에 가격, 용량, 100g당 가격이 표시된다.

### 9.2 라떼용 원두 검색과 비교

Given 사용자가 홈 화면에 있다.  
When `라떼에 좋은 원두`를 검색한다.  
And 바디감 묵직함 필터를 적용한다.  
And 원두 3개를 비교함에 추가한다.  
Then 비교 화면에서 3개 원두의 가격, 100g당 가격, 산미, 바디감, 컵노트가 표시된다.

### 9.3 가격순 탐색

Given 사용자가 검색 결과 화면에 있다.  
When `100g당 가격 낮은 순` 정렬을 선택한다.  
Then price_per_100g이 낮은 원두부터 표시된다.

### 9.4 상세에서 판매처 이동

Given 사용자가 원두 상세 화면에 있다.  
When 판매처 이동 버튼을 클릭한다.  
Then 외부 판매처가 새 탭으로 열린다.  
And outbound_clicked 이벤트가 전송된다.

### 9.5 빈 결과 복구

Given 사용자가 검색 결과 화면에 있다.  
When 결과가 없는 필터 조합을 적용한다.  
Then 빈 결과 안내가 표시된다.  
When 필터 초기화 버튼을 클릭한다.  
Then 검색어는 유지되고 필터가 제거된다.

## 10. 화면 QA 체크리스트

### 10.1 홈

- 검색창이 보인다.
- 추천 검색어가 보인다.
- 주요 카테고리 진입점이 보인다.
- 랭킹 진입점이 보인다.
- 빈 검색어 제출 시 전체 검색 결과로 이동한다.

### 10.2 검색 결과

- 검색 조건 요약이 보인다.
- 결과 수가 보인다.
- 필터 패널이 동작한다.
- 정렬 선택이 동작한다.
- 원두 카드에 원두명, 로스터리명, 가격, 용량, 100g당 가격이 보인다.
- 원두 카드에 산미와 바디감 요약이 보인다.
- 더 보기 버튼이 다음 결과를 불러온다.
- 품절 원두는 품절 상태로 표시된다.

### 10.3 상세

- 원두명과 로스터리명이 보인다.
- 가격, 용량, 100g당 가격이 보인다.
- 원산지, 로스팅, 가공 방식이 보인다.
- 맛 프로필이 보인다.
- 컵노트가 보인다.
- 추천 추출 방식이 보인다.
- 유사 원두가 보인다.
- 판매처 이동 버튼이 동작한다.

### 10.4 비교

- 비교함에 담긴 원두 수가 보인다.
- 최대 4개 원두를 비교할 수 있다.
- 원두 제거가 동작한다.
- 전체 비우기가 동작한다.
- 가격과 100g당 가격을 비교할 수 있다.
- 산미, 단맛, 쓴맛, 바디감이 비교된다.

### 10.5 카테고리와 랭킹

- 카테고리 설명이 보인다.
- 카테고리 기본 조건 chip이 보인다.
- 랭킹 산정 기준이 보인다.
- 랭킹 번호가 보인다.
- 목록에서 상세, 비교, 판매처 이동이 가능하다.

## 11. 반응형 테스트

테스트 뷰포트:

| 기기 | 크기 |
| --- | --- |
| 모바일 | 390x844 |
| 태블릿 | 768x1024 |
| 데스크톱 | 1440x900 |

체크리스트:

- 모바일에서 원두 카드가 1열로 표시된다.
- 태블릿에서 카드가 2열 또는 적절한 레이아웃으로 표시된다.
- 데스크톱에서 필터 사이드바와 카드 그리드가 함께 표시된다.
- 모바일 필터 drawer가 열리고 닫힌다.
- 버튼 텍스트가 영역 밖으로 넘치지 않는다.
- 원두명이 길어도 카드 레이아웃이 깨지지 않는다.
- 비교 화면이 모바일에서 가로 스크롤 또는 세로 비교로 읽을 수 있다.

## 12. 접근성 테스트

체크리스트:

- 모든 버튼은 키보드로 포커스 가능하다.
- 검색 입력창은 label 또는 접근 가능한 이름을 가진다.
- 아이콘 버튼은 접근 가능한 이름을 가진다.
- 필터 drawer는 열릴 때 포커스가 내부로 이동한다.
- 필터 drawer는 Escape 또는 닫기 버튼으로 닫힌다.
- 맛 점수는 색상 외 숫자와 라벨로도 이해할 수 있다.
- 원두 이미지는 대체 텍스트를 가진다.
- 판매처 이동 버튼의 새 탭 동작이 사용자에게 이해 가능하다.
- 색 대비가 기본 접근성 기준을 만족한다.

## 16. 출시 전 수용 기준

MVP 베타 출시 전 다음 기준을 만족해야 한다.

필수:

- P0 기능 테스트가 모두 통과한다.
- 데이터 검증 테스트가 모두 통과한다.
- 검색, 필터, 정렬 핵심 E2E가 통과한다.
- 상세 화면과 비교 화면 E2E가 통과한다.
- 판매처 이동 이벤트가 기록된다.
- 모바일, 태블릿, 데스크톱 주요 화면이 깨지지 않는다.
- 검색 결과 첫 화면이 2초 이내 표시된다.
- 접근성 치명 오류가 없다.

허용 가능한 잔여 이슈:

- 필터 count가 없거나 일부 부정확한 경우
- 평점/리뷰 데이터가 없는 경우
- 유사 원두 추천 품질이 완벽하지 않은 경우
- 일부 원두의 선택 필드가 `정보 없음`으로 표시되는 경우

출시 차단 이슈:

- 검색이 동작하지 않음
- 원두 상세 접근 불가
- 가격 또는 100g당 가격이 잘못 표시됨
- 비교함 추가/제거 불가
- 모바일에서 핵심 화면 사용 불가
- 필수 데이터 누락이 사용자 화면 오류로 이어짐
