# 커피 원두 검색 엔진 PRD

문서 버전: v0.1  
작성일: 2026-05-08  
제품명: Bean Finder, 가칭  
문서 목적: 커피 원두별 맛, 특징, 가격을 비교하고 사용자의 취향에 맞는 원두를 찾도록 돕는 검색 서비스의 MVP 요구사항을 정의한다.

## 1. 배경

한국의 커피 소비자는 커피에 대한 관심과 선택지가 모두 높지만, 원두 구매 과정은 여전히 어렵다. 같은 원두라도 판매처마다 용량과 가격 표기가 다르고, 산미, 바디감, 컵노트, 로스팅 정도 같은 정보는 초보자가 이해하기 어렵다.

이 서비스는 원두 정보를 표준화하고, 맛과 가격을 기준으로 쉽게 탐색할 수 있는 커피 원두 검색 엔진을 제공한다.

## 2. 문제 정의

사용자는 원두를 구매할 때 다음 문제를 겪는다.

- 원두의 맛 설명이 전문 용어 위주라 이해하기 어렵다.
- 산미, 바디감, 단맛, 쓴맛 등 취향 기준으로 검색하기 어렵다.
- 200g, 250g, 500g 등 용량이 달라 가격 비교가 어렵다.
- 핸드드립, 에스프레소, 라떼, 콜드브루 등 추출 방식별 적합 원두를 찾기 어렵다.
- 여러 로스터리의 원두를 한 화면에서 비교하기 어렵다.
- 입문자는 어떤 원두가 실패 가능성이 낮은지 판단하기 어렵다.

## 3. 제품 목표

MVP의 목표는 사용자가 자신의 취향과 예산에 맞는 원두 후보를 빠르게 찾고 비교할 수 있게 만드는 것이다.

핵심 목표는 다음과 같다.

- 원두 정보를 표준화된 데이터로 정리한다.
- 맛, 가격, 원산지, 로스팅, 추출 방식 기준으로 검색할 수 있게 한다.
- 원두별 산미, 바디감, 단맛, 쓴맛, 향미 특성을 시각적으로 비교한다.
- 용량이 다른 상품도 100g당 가격으로 비교한다.
- 입문자도 이해할 수 있는 쉬운 표현과 전문 정보를 함께 제공한다.

## 4. 비전

Bean Finder는 국내 커피 소비자를 위한 취향 기반 원두 탐색 서비스가 된다. 장기적으로는 사용자의 취향 기록, 리뷰, 구매 이력, 가격 변동 데이터를 기반으로 개인화 추천과 원두 구매 의사결정을 지원한다.

## 5. 타깃 사용자

### 5.1 입문자

특징:

- 커피 원두를 직접 구매한 경험이 적다.
- 산미, 바디감, 컵노트 같은 용어가 익숙하지 않다.
- 실패 가능성이 낮은 원두를 찾고 싶어 한다.

핵심 니즈:

- "신맛이 적은 원두"
- "고소한 원두"
- "입문자에게 무난한 원두"
- "2만원 이하로 살 수 있는 원두"

### 5.2 홈카페 사용자

특징:

- 집에서 핸드드립, 에스프레소 머신, 모카포트, 콜드브루 등을 사용한다.
- 여러 로스터리의 원두를 비교하며 구매한다.
- 가격 대비 만족도와 재구매 가능성을 중요하게 본다.

핵심 니즈:

- 추출 방식에 맞는 원두 추천
- 100g당 가격 비교
- 라떼용, 드립용, 콜드브루용 필터
- 비슷한 맛의 대체 원두 탐색

### 5.3 커피 취향이 뚜렷한 사용자

특징:

- 원산지, 품종, 가공 방식, 로스팅 정도에 관심이 있다.
- 컵노트와 맛 프로필을 기준으로 원두를 고른다.
- 가격보다 취향 적합도와 원두 특성을 중요하게 볼 수 있다.

핵심 니즈:

- 원산지, 품종, 가공 방식 필터
- 컵노트 기반 검색
- 맛 프로필 상세 비교
- 고가 원두와 스페셜티 원두 탐색

## 6. 핵심 사용자 시나리오

### 6.1 입문자가 신맛이 적은 원두를 찾는 경우

1. 사용자가 홈 화면에서 "신맛 적은 원두" 카테고리를 선택한다.
2. 검색 결과에서 산미 낮음, 고소함, 중간 이하 가격대 필터가 적용된다.
3. 사용자는 100g당 가격과 맛 프로필을 확인한다.
4. 원두 상세 화면에서 컵노트, 로스팅, 추천 추출 방식을 본다.
5. 마음에 드는 원두를 판매처로 이동해 구매한다.

### 6.2 홈카페 사용자가 라떼용 원두를 찾는 경우

1. 사용자가 검색창에 "라떼에 좋은 묵직한 원두"를 입력한다.
2. 서비스는 바디감 높음, 산미 낮음, 미디엄다크 이상 로스팅, 에스프레소 적합 원두를 우선 노출한다.
3. 사용자는 가격순 또는 평점순으로 정렬한다.
4. 2개 이상의 원두를 비교함에 담아 비교한다.
5. 가격과 맛 특성을 비교한 뒤 판매처로 이동한다.

### 6.3 커피 애호가가 특정 컵노트 원두를 찾는 경우

1. 사용자가 "베리, 플로럴, 라이트 로스팅" 조건을 선택한다.
2. 서비스는 에티오피아, 케냐 등 조건에 맞는 원두를 보여준다.
3. 사용자는 가공 방식과 산미 강도를 추가 필터링한다.
4. 상세 화면에서 유사 원두를 함께 확인한다.

## 8. 기능 요구사항

### 8.1 홈 화면

목적: 사용자가 원하는 원두 탐색을 시작하게 한다.

필수 요소:

- 검색창
- 추천 카테고리
- 인기 필터 바로가기
- 가격대별 랭킹 진입점
- 입문자 추천 원두 진입점

추천 카테고리:

- 산미 있는 원두
- 신맛 적은 원두
- 고소한 원두
- 묵직한 바디감
- 과일향/베리 계열
- 초콜릿/견과류 계열
- 라떼에 좋은 원두
- 핸드드립 입문용
- 가성비 원두
- 디카페인 원두

### 8.2 검색

목적: 사용자가 키워드와 조건으로 원두를 찾게 한다.

검색 대상:

- 원두명
- 로스터리명
- 원산지
- 지역
- 품종
- 가공 방식
- 컵노트
- 추출 방식
- 쉬운 맛 표현 태그

검색 예시:

- 산미 낮고 고소한 원두
- 라떼에 좋은 원두
- 2만원 이하 에티오피아 원두
- 초콜릿 향 나는 다크로스팅 원두
- 핸드드립 입문용 원두

### 8.3 필터

필수 필터:

| 필터 | 옵션 |
| --- | --- |
| 가격대 | 1만원 이하, 1만-2만원, 2만-3만원, 3만원 이상 |
| 100g당 가격 | 직접 범위 입력 또는 슬라이더 |
| 산미 | 낮음, 보통, 높음 |
| 바디감 | 가벼움, 보통, 묵직함 |
| 로스팅 | 라이트, 미디엄, 미디엄다크, 다크 |
| 원산지 | 국가 단위 |
| 컵노트 | 과일, 베리, 꽃, 견과류, 초콜릿, 캐러멜, 허브 등 |
| 추출 방식 | 핸드드립, 에스프레소, 라떼, 콜드브루 |
| 디카페인 | 포함, 제외 |
| 판매 상태 | 판매 중, 품절 제외 |

### 8.4 정렬

필수 정렬:

- 추천순
- 가격 낮은 순
- 100g당 가격 낮은 순
- 평점 높은 순
- 리뷰 많은 순
- 산미 강한 순
- 바디감 강한 순
- 최신 등록순

MVP 추천순 기준:

```text
추천 점수 =
맛 조건 일치도 45%
+ 가격 적합도 20%
+ 데이터 완성도 15%
+ 사용자 평점 10%
+ 리뷰 수 10%
```

사용자 평점과 리뷰 수 데이터가 없을 경우, 해당 항목은 데이터 완성도와 조건 일치도로 대체한다.

원두 카드에 표시할 정보:

- 로스팅 정도
- 산미/바디감 요약
- 대표 컵노트 2-3개
- 추천 추출 방식
- 비교함 추가 버튼
- 판매처 이동 버튼

검색 결과 상단에 표시할 정보:

- 검색 조건 요약
- 검색 결과 수
- 적용된 필터

필수 정보:

- 원산지

- 농장 또는 생산자, 데이터가 있을 경우

- 가격

- 컵노트

### 8.7 원두 비교

목적: 여러 원두를 한 화면에서 비교해 구매 결정을 돕는다.

요구사항:

- 최대 4개 원두 비교
- 비교 항목은 행, 원두는 열로 표시
- 가격, 용량, 100g당 가격 비교
- 산미, 단맛, 쓴맛, 바디감, 향미 강도 비교
- 원산지, 로스팅, 컵노트, 추출 방식 비교
- 판매처 이동 버튼 제공

### 8.8 카테고리 페이지

목적: 검색어를 모르는 사용자도 취향 기반으로 탐색하게 한다.

필수 카테고리:

- 신맛 적은 원두
- 산미 있는 원두
- 고소한 원두
- 묵직한 원두
- 라떼용 원두
- 핸드드립용 원두
- 2만원 이하 원두
- 가성비 원두
- 디카페인 원두

각 카테고리 페이지에는 다음을 제공한다.

- 카테고리 설명
- 대표 필터
- 추천 원두 목록
- 가격순/추천순 정렬

랭킹 유형:

- 라떼 추천 원두

### 9.2 가격 정규화

모든 원두는 100g당 가격을 계산한다.

```text
100g당 가격 = 판매 가격 / 용량(g) * 100
```

예시:

```text
18,000원 / 200g * 100 = 9,000원
```

### 9.3 맛 점수 기준

맛 점수는 1-5 범위로 정규화한다.

| 점수 | 의미 |
| --- | --- |
| 1 | 매우 낮음 |
| 2 | 낮음 |
| 3 | 보통 |
| 4 | 높음 |
| 5 | 매우 높음 |

초기 데이터는 로스터리의 설명, 컵노트, 로스팅 정도를 바탕으로 운영자가 태깅한다. 이후 사용자 리뷰와 평점을 반영해 보정할 수 있다.

## 10. 콘텐츠 정책

### 10.1 쉬운 표현과 전문 표현 병행

서비스는 전문 용어를 그대로 노출하되, 입문자가 이해할 수 있는 쉬운 표현을 함께 제공한다.

예시:

| 전문 표현 | 쉬운 표현 |
| --- | --- |
| 산미 낮음 | 신맛이 적음 |
| 바디감 높음 | 묵직함 |
| 라이트 로스팅 | 밝고 산뜻함 |
| 다크 로스팅 | 진하고 쌉쌀함 |
| 너티 | 고소함 |
| 플로럴 | 꽃향 |

### 10.2 데이터 신뢰도

데이터 출처는 다음 우선순위로 관리한다.

1. 로스터리 공식 상품 정보
2. 판매처 상품 정보
3. 운영자 태깅
4. 사용자 리뷰 기반 보정

운영자 태깅 항목은 내부적으로 출처를 구분해 저장한다.

## 11. 성공 지표

MVP 출시 후 확인할 핵심 지표:

| 지표 | 목표 |
| --- | --- |
| 검색 실행률 | 방문자의 40% 이상 |
| 필터 사용률 | 검색 사용자의 30% 이상 |
| 상세 화면 진입률 | 검색 결과 사용자의 25% 이상 |
| 비교 기능 사용률 | 상세/검색 사용자의 10% 이상 |
| 판매처 이동률 | 상세 화면 사용자의 15% 이상 |
| 평균 탐색 시간 | 2분 이상 |
| 재방문율 | 20% 이상 |

정량 지표 외에도 다음 정성 지표를 본다.

- 사용자가 자신의 취향에 맞는 원두를 찾았다고 느끼는가
- 맛 표현이 이해하기 쉬운가
- 가격 비교가 구매 결정에 도움이 되는가
- 원두 비교 화면이 충분히 명확한가

## 12. 비기능 요구사항

### 12.1 성능

- 검색 결과 첫 화면은 2초 이내 표시한다.
- 필터 변경 후 결과 갱신은 1초 이내를 목표로 한다.
- 원두 카드 이미지는 지연 로딩한다.

### 12.2 사용성

- 모바일 화면에서 필터와 정렬을 쉽게 조작할 수 있어야 한다.
- 입문자를 위한 쉬운 맛 표현을 우선 표시하고, 상세 화면에서 전문 정보를 함께 제공한다.
- 가격은 원화 기준으로 표시한다.
- 100g당 가격은 모든 목록과 비교 화면에 노출한다.

### 12.3 접근성

- 색상만으로 맛 점수를 구분하지 않는다.
- 버튼과 필터는 키보드 접근이 가능해야 한다.
- 이미지에는 대체 텍스트를 제공한다.

### 12.4 데이터 관리

- 원두 정보는 마지막 확인일을 저장한다.
- 판매 중단 또는 품절 상태를 표시할 수 있어야 한다.
- 가격과 용량 변경에 대비해 데이터 갱신 시점을 기록한다.

## 13. 관리자 요구사항

MVP에서는 별도 관리자 페이지를 만들지 않고, 운영자가 구조화된 데이터 파일 또는 데이터베이스에 직접 등록하는 방식을 허용한다.

관리자가 등록해야 하는 정보:

- 원두 기본 정보
- 가격과 용량
- 맛 점수
- 컵노트
- 추천 추출 방식
- 판매처 링크
- 데이터 출처
- 마지막 확인일

추후 관리자 페이지 요구사항:

- 원두 등록/수정/삭제
- 판매 상태 관리
- 가격 변경 이력
- 데이터 완성도 체크
- 로스터리별 원두 관리

## 14. MVP 데이터 수집 기준

초기 데이터 목표:

- 로스터리 20-50곳
- 원두 300-500개
- 국내 판매 중인 원두 중심
- 가격과 용량이 명확한 상품 우선
- 원산지, 로스팅, 컵노트가 확인 가능한 상품 우선

초기에는 완벽한 자동화보다 정확한 수작업 태깅을 우선한다.

## 19. 오픈 질문

추가 결정이 필요한 사항:

- 초기 데이터 수집 대상 로스터리는 어디까지 포함할 것인가
- 데이터는 수작업 CSV로 시작할 것인가, 데이터베이스에 바로 넣을 것인가
- 평점과 리뷰 수는 외부 판매처 기준으로 가져올 것인가, 자체 리뷰 출시 후 반영할 것인가
- 추천순에서 가격과 맛 조건 중 무엇을 더 강하게 반영할 것인가
- 입문자용 쉬운 표현의 기준을 어떻게 정할 것인가
- 판매처 이동은 제휴 링크 기반으로 시작할 것인가, 일반 링크로 시작할 것인가

## 20. 우선순위 요약

가장 먼저 구현해야 할 핵심은 다음 3가지다.

1. 원두 데이터 모델
2. 검색 결과 화면
3. 원두 상세 및 비교 화면

이 3가지가 완성되면 서비스의 핵심 가치인 "취향과 가격 기준으로 원두를 비교해 찾는 경험"을 검증할 수 있다.
