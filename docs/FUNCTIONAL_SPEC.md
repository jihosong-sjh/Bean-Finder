# 커피 원두 검색 엔진 기능 명세서

문서 버전: v0.1  
작성일: 2026-05-08  
제품명: Bean Finder, 가칭  
연관 문서: [PRD.md](./PRD.md)  
문서 목적: PRD에서 정의한 MVP 범위를 실제 개발 가능한 기능 단위로 분해하고, 각 기능의 동작 규칙, 입력, 출력, 예외 상태, 수용 기준을 정의한다.

## 1. 범위

이 문서는 Bean Finder MVP의 사용자-facing 기능과 운영자용 데이터 등록 흐름을 다룬다.

MVP 포함 기능:

- 홈 탐색
- 키워드 검색
- 필터 검색
- 정렬
- 검색 결과 목록
- 원두 상세
- 원두 비교
- 취향 기반 카테고리
- 랭킹
- 판매처 이동
- 기본 데이터 등록 및 검증

MVP 제외 기능:

- 회원가입과 로그인
- 자체 결제
- 사용자 리뷰 작성
- 개인화 추천
- 가격 변동 알림
- 자동 크롤링
- 관리자 웹 화면
- 모바일 네이티브 앱

## 2. 용어 정의

| 용어 | 정의 |
| --- | --- |
| 원두 | 서비스에서 검색, 비교, 추천의 대상이 되는 커피 원두 상품 |
| 로스터리 | 원두를 로스팅하고 판매하는 브랜드 또는 업체 |
| 컵노트 | 원두에서 느껴지는 향미 표현. 예: 베리, 초콜릿, 견과류, 꽃향 |
| 산미 | 커피에서 느껴지는 밝고 상큼한 맛의 정도 |
| 바디감 | 커피의 질감, 무게감, 입안에서 느껴지는 밀도 |
| 로스팅 정도 | 원두를 볶은 정도. 라이트, 미디엄, 미디엄다크, 다크 |
| 추출 방식 | 원두를 마시는 방식. 핸드드립, 에스프레소, 라떼, 콜드브루 등 |
| 100g당 가격 | 용량이 다른 상품을 비교하기 위해 정규화한 가격 |
| 비교함 | 사용자가 비교할 원두를 임시로 담아두는 클라이언트 상태 |
| 추천순 | 검색 조건, 가격, 데이터 완성도, 평점, 리뷰 수를 반영한 기본 정렬 |

## 3. 사용자 역할

| 역할 | 설명 | MVP 권한 |
| --- | --- | --- |
| 방문자 | 로그인하지 않은 일반 사용자 | 검색, 필터, 상세 조회, 비교, 판매처 이동 |
| 운영자 | 원두 데이터를 등록하고 관리하는 내부 사용자 | 구조화된 데이터 파일 또는 DB 직접 관리 |

MVP에서는 방문자 계정을 만들지 않는다. 찜, 최근 본 원두, 개인화 추천은 구현하지 않는다.

## 5. 공통 규칙

### 5.1 가격 표시

- 모든 가격은 원화 기준으로 표시한다.
- 목록, 상세, 비교 화면에서는 판매 가격과 100g당 가격을 모두 표시한다.
- 100g당 가격은 다음 공식으로 계산한다.

```text
price_per_100g = price / weight_g * 100
```

- 계산 결과는 원 단위로 반올림한다.
- 가격 또는 용량이 누락된 원두는 검색 결과에 노출하지 않는다.

### 5.2 맛 점수 표시

- 맛 점수는 1-5 범위의 정수로 표시한다.
- 산미, 단맛, 쓴맛, 바디감은 필수 표시한다.
- 향미 강도와 밸런스는 데이터가 있을 때만 표시한다.
- 점수는 색상만으로 표현하지 않고 숫자, 라벨, 그래프 중 최소 2개 방식으로 이해할 수 있게 한다.

맛 점수 라벨:

| 점수 | 라벨 |
| --- | --- |
| 1 | 매우 낮음 |
| 2 | 낮음 |
| 3 | 보통 |
| 4 | 높음 |
| 5 | 매우 높음 |

### 5.3 쉬운 표현 매핑

사용자에게는 전문 표현과 쉬운 표현을 함께 제공한다.

| 조건 | 쉬운 표현 |
| --- | --- |
| acidity 1-2 | 신맛 적음 |
| acidity 4-5 | 산미 있음 |
| body 1-2 | 가벼움 |
| body 4-5 | 묵직함 |
| bitterness 1-2 | 쓴맛 적음 |
| sweetness 4-5 | 단맛 좋음 |
| roast_level = dark | 진하고 쌉쌀함 |
| tasting_notes에 nut 또는 nutty 계열 포함 | 고소함 |
| tasting_notes에 chocolate 계열 포함 | 초콜릿 느낌 |
| tasting_notes에 berry 계열 포함 | 베리향 |
| tasting_notes에 floral 계열 포함 | 꽃향 |

### 5.4 판매 상태

- 기본 검색 결과에는 판매 중인 원두만 노출한다.
- 사용자가 품절 포함 필터를 선택한 경우 품절 원두도 노출할 수 있다.
- 품절 원두에는 판매처 이동 버튼 대신 "품절" 상태를 표시한다.

### 5.5 상태 유지

- 검색어, 필터, 정렬은 URL 쿼리 파라미터로 표현할 수 있어야 한다.
- 사용자가 검색 결과에서 상세 화면으로 이동했다가 뒤로 가기 했을 때 기존 검색 조건이 유지되어야 한다.
- 비교함은 MVP에서 브라우저 로컬 상태로 유지한다.

## 6. FS-001 홈 탐색

### 6.1 목적

사용자가 검색어를 직접 입력하거나 취향 기반 카테고리를 선택해 원두 탐색을 시작할 수 있게 한다.

### 6.2 진입 경로

- 기본 URL 진입
- 로고 클릭
- 브라우저 주소 직접 입력

### 6.3 화면 요소

- 검색 입력창
- 검색 실행 버튼
- 추천 카테고리 목록
- 가격대별 랭킹 진입점
- 입문자 추천 진입점

### 6.4 동작 규칙

- 검색창에 키워드를 입력하고 실행하면 검색 결과 화면으로 이동한다.
- 빈 검색어로 검색하면 전체 원두 목록을 추천순으로 보여준다.
- 추천 카테고리를 선택하면 해당 카테고리의 기본 필터가 적용된 검색 결과 또는 카테고리 화면으로 이동한다.
- 가격대별 랭킹을 선택하면 랭킹 화면으로 이동한다.

### 6.5 수용 기준

- 사용자는 홈에서 1번의 검색 실행으로 검색 결과를 볼 수 있다.
- 사용자는 홈에서 주요 카테고리 중 하나를 선택해 원두 목록을 볼 수 있다.
- 빈 검색어 제출 시 오류가 아닌 전체 탐색 결과를 제공한다.

## 7. FS-002 키워드 검색

### 7.1 목적

사용자가 원두명, 로스터리, 원산지, 컵노트, 맛 표현 등 자연스러운 키워드로 원두를 찾을 수 있게 한다.

### 7.2 검색 대상 필드

- name
- roastery_name
- origin_country
- origin_region
- farm
- variety
- process
- roast_level
- tasting_notes
- recommended_brew_methods
- easy_taste_tags

### 7.3 입력

| 항목 | 형식 | 필수 여부 |
| --- | --- | --- |
| query | 문자열 | 선택 |

검색어 예시:

- 신맛 적은 원두
- 산미 있는 에티오피아
- 라떼에 좋은 원두
- 초콜릿 향 다크로스팅
- 2만원 이하 원두

### 7.4 동작 규칙

- 검색어 앞뒤 공백은 제거한다.
- 대소문자는 구분하지 않는다.
- 한글, 영문, 숫자를 검색할 수 있어야 한다.
- 검색어가 여러 단어인 경우 가능한 조건을 조합해 결과를 찾는다.
- "신맛 적은"은 acidity 1-2 조건으로 해석한다.
- "산미 있는"은 acidity 4-5 조건으로 해석한다.
- "묵직한"은 body 4-5 조건으로 해석한다.
- "고소한"은 nutty 계열 컵노트 또는 easy_taste_tags에 고소함이 있는 원두로 해석한다.
- "라떼"는 recommended_brew_methods에 latte 또는 espresso가 있는 원두를 우선한다.
- "2만원 이하"는 price <= 20000 조건으로 해석한다.

### 7.5 결과 우선순위

동일한 검색어에서 결과가 여러 개일 경우 다음 순서로 점수를 부여한다.

1. 원두명 직접 일치
2. 로스터리명 직접 일치
3. 컵노트 일치
4. 쉬운 맛 표현 일치
5. 원산지 또는 지역 일치
6. 추출 방식 일치
7. 로스팅 정도 일치

### 7.6 예외 상태

- 검색어가 너무 긴 경우 최대 허용 길이까지만 처리하고 안내 문구를 표시한다.
- 인식할 수 없는 검색어라도 전체 텍스트 검색 결과를 제공한다.
- 검색 결과가 없으면 빈 결과 상태를 표시한다.

### 7.7 수용 기준

- "신맛 적은 원두" 검색 시 산미 점수가 낮은 원두가 우선 노출된다.
- "라떼 원두" 검색 시 라떼 또는 에스프레소에 적합한 원두가 우선 노출된다.
- "2만원 이하" 검색 시 판매 가격이 20,000원 이하인 원두만 노출된다.
- 검색어가 비어 있어도 전체 원두 목록을 볼 수 있다.

## 8. FS-003 필터 검색

### 8.1 목적

사용자가 가격, 맛, 원산지, 로스팅, 컵노트, 추출 방식 등 명확한 조건으로 결과를 좁힐 수 있게 한다.

### 8.2 필터 목록

| 필터 | 파라미터 | 옵션 |
| --- | --- | --- |
| 가격대 | price_range | under_10000, 10000_20000, 20000_30000, over_30000 |
| 100g당 가격 | price_per_100g_min, price_per_100g_max | 숫자 범위 |
| 산미 | acidity | low, medium, high |
| 바디감 | body | light, medium, heavy |
| 로스팅 | roast_level | light, medium, medium_dark, dark |
| 원산지 | origin_country | 국가 코드 또는 국가명 |
| 컵노트 | tasting_notes | 복수 선택 |
| 추출 방식 | brew_method | hand_drip, espresso, latte, cold_brew |
| 디카페인 | is_decaf | true, false |
| 판매 상태 | availability | available_only, include_sold_out |

### 8.3 필터 매핑

산미:

| 옵션 | 조건 |
| --- | --- |
| 낮음 | acidity 1-2 |
| 보통 | acidity 3 |
| 높음 | acidity 4-5 |

바디감:

| 옵션 | 조건 |
| --- | --- |
| 가벼움 | body 1-2 |
| 보통 | body 3 |
| 묵직함 | body 4-5 |

가격대:

| 옵션 | 조건 |
| --- | --- |
| 1만원 이하 | price <= 10000 |
| 1만-2만원 | 10000 < price <= 20000 |
| 2만-3만원 | 20000 < price <= 30000 |
| 3만원 이상 | price > 30000 |

### 8.4 동작 규칙

- 서로 다른 필터 그룹은 AND 조건으로 적용한다.
- 같은 필터 그룹 안에서 복수 선택이 가능한 항목은 OR 조건으로 적용한다.
- 필터를 변경하면 결과 수와 목록이 갱신된다.
- 필터 적용 후 정렬 조건은 유지된다.
- 필터 초기화를 누르면 검색어를 제외한 모든 필터가 제거된다.
- 전체 초기화를 누르면 검색어, 필터, 정렬이 기본값으로 돌아간다.

### 8.5 예외 상태

- 필터 조건에 맞는 원두가 없으면 빈 결과 상태를 표시한다.
- 100g당 가격 최소값이 최대값보다 크면 적용하지 않고 입력 오류를 표시한다.
- 사용할 수 없는 필터 값이 URL에 포함되면 해당 값은 무시한다.

### 8.6 수용 기준

- 사용자는 산미 낮음과 바디감 묵직함을 동시에 적용할 수 있다.
- 사용자는 컵노트를 여러 개 선택할 수 있다.
- 필터 초기화 시 검색어는 유지된다.
- 전체 초기화 시 기본 검색 결과로 돌아간다.

## 9. FS-004 검색 결과 정렬

### 9.1 목적

사용자가 검색 결과를 가격, 맛 특성, 평점, 최신 등록 등 원하는 기준으로 정렬할 수 있게 한다.

### 9.2 정렬 옵션

| 정렬 | 파라미터 | 기준 |
| --- | --- | --- |
| 추천순 | recommended | 추천 점수 내림차순 |
| 가격 낮은 순 | price_asc | price 오름차순 |
| 100g당 가격 낮은 순 | price_per_100g_asc | price_per_100g 오름차순 |
| 평점 높은 순 | rating_desc | rating 내림차순 |
| 리뷰 많은 순 | review_count_desc | review_count 내림차순 |
| 산미 강한 순 | acidity_desc | acidity 내림차순 |
| 바디감 강한 순 | body_desc | body 내림차순 |
| 최신 등록순 | newest | created_at 내림차순 |

### 9.3 추천순 점수

MVP 추천순은 다음 점수를 기준으로 한다.

```text
recommendation_score =
taste_match_score * 0.45
+ price_fit_score * 0.20
+ data_completeness_score * 0.15
+ rating_score * 0.10
+ review_count_score * 0.10
```

평점과 리뷰 수가 없는 경우:

```text
recommendation_score =
taste_match_score * 0.55
+ price_fit_score * 0.25
+ data_completeness_score * 0.20
```

### 9.4 동작 규칙

- 기본 정렬은 추천순이다.
- 정렬을 변경해도 검색어와 필터는 유지된다.
- 정렬 기준에 필요한 데이터가 없는 원두는 해당 정렬에서 하위에 배치한다.
- 동일 점수일 경우 데이터 완성도가 높은 원두를 우선한다.
- 데이터 완성도도 같으면 원두명 오름차순으로 정렬한다.

### 9.5 수용 기준

- 가격 낮은 순 선택 시 price가 낮은 원두부터 표시된다.
- 100g당 가격 낮은 순 선택 시 용량 차이를 반영한 price_per_100g 기준으로 표시된다.
- 정렬 변경 후에도 적용된 필터는 유지된다.

## 10. FS-005 검색 결과 목록

### 10.1 목적

사용자가 검색과 필터 조건에 맞는 원두를 카드 형태로 빠르게 비교하고 상세 화면으로 이동할 수 있게 한다.

### 10.2 목록 상단 정보

- 검색어
- 적용된 필터
- 결과 수
- 정렬 선택
- 필터 초기화 버튼

### 10.3 원두 카드 정보

필수 표시:

- 원두명
- 로스터리명
- 대표 이미지 또는 기본 이미지
- 가격
- 용량
- 100g당 가격
- 원산지
- 로스팅 정도
- 산미 요약
- 바디감 요약
- 대표 컵노트 최대 3개
- 추천 추출 방식 최대 2개
- 비교함 추가 버튼
- 상세 보기 진입
- 판매처 이동 버튼

### 10.4 동작 규칙

- 카드 클릭 시 원두 상세 화면으로 이동한다.
- 비교함 추가 버튼 클릭 시 해당 원두를 비교함에 추가한다.
- 이미 비교함에 있는 원두는 "비교함에 있음" 상태로 표시한다.
- 판매처 이동 버튼 클릭 시 외부 판매처 링크를 새 탭으로 연다.
- 품절 원두는 카드에 품절 상태를 표시한다.
- 이미지가 없으면 기본 이미지를 표시한다.

### 10.5 페이지네이션 또는 무한 스크롤

MVP 기본안:

- 첫 화면에 20개 원두를 표시한다.
- 더 보기 버튼으로 다음 20개를 불러온다.

추후 확장:

- 무한 스크롤을 도입할 수 있다.

### 10.6 수용 기준

- 사용자는 검색 결과에서 원두의 가격과 100g당 가격을 동시에 볼 수 있다.
- 사용자는 원두 카드에서 바로 비교함에 추가할 수 있다.
- 사용자는 원두 카드에서 상세 화면과 판매처로 이동할 수 있다.

## 11. FS-006 원두 상세 조회

### 11.1 목적

사용자가 특정 원두의 맛, 가격, 특징, 추천 추출 방식, 판매처 정보를 상세히 확인할 수 있게 한다.

### 11.2 진입 경로

- 검색 결과 카드 클릭
- 비교 화면의 원두명 클릭
- 카테고리 또는 랭킹 목록의 원두 클릭
- 직접 URL 접근

### 11.3 표시 정보

기본 정보:

- 원두명
- 로스터리명
- 대표 이미지
- 원산지
- 지역
- 농장 또는 생산자
- 품종
- 가공 방식
- 로스팅 정도
- 디카페인 여부
- 판매 상태
- 마지막 정보 확인일

가격 정보:

- 판매 가격
- 용량
- 100g당 가격
- 판매처 링크

맛 정보:

- 컵노트
- 산미
- 단맛
- 쓴맛
- 바디감
- 향미 강도
- 밸런스
- 쉬운 맛 표현 태그

추천 정보:

- 추천 추출 방식
- 유사 원두
- 같은 로스터리의 다른 원두

### 11.4 동작 규칙

- 없는 데이터는 "정보 없음"으로 표시하되 화면 레이아웃이 깨지지 않아야 한다.
- 필수 데이터가 누락된 원두는 상세 페이지 접근 전에 데이터 검증에서 차단한다.
- 판매처 이동 버튼 클릭 시 새 탭에서 product_url을 연다.
- 비교함 추가 버튼을 제공한다.
- 유사 원두는 맛 점수, 컵노트, 로스팅, 원산지 유사도를 기준으로 최대 6개 표시한다.

### 11.5 유사 원두 기준

MVP 유사도 기준:

```text
similarity_score =
same_origin_score
+ same_roast_level_score
+ taste_profile_distance_score
+ tasting_note_overlap_score
```

가중치:

| 항목 | 가중치 |
| --- | --- |
| 맛 프로필 거리 | 40% |
| 컵노트 겹침 | 30% |
| 로스팅 동일 여부 | 20% |
| 원산지 동일 여부 | 10% |

### 11.6 수용 기준

- 사용자는 상세 화면에서 원두의 맛 프로필을 확인할 수 있다.
- 사용자는 상세 화면에서 100g당 가격을 확인할 수 있다.
- 사용자는 상세 화면에서 판매처로 이동할 수 있다.
- 사용자는 상세 화면에서 해당 원두를 비교함에 추가할 수 있다.

## 12. FS-007 원두 비교

### 12.1 목적

사용자가 최대 4개 원두를 한 화면에서 비교해 구매 결정을 쉽게 하도록 한다.

### 12.2 진입 경로

- 검색 결과 카드의 비교함 추가
- 상세 화면의 비교함 추가
- 비교함 버튼 클릭

### 12.3 비교 가능 개수

- 최소 2개부터 비교 화면의 가치가 있다.
- 최대 4개까지 비교할 수 있다.
- 원두 1개만 담긴 경우 비교 화면은 표시하되 추가 안내를 제공한다.

### 12.4 비교 항목

| 항목 | 표시 여부 |
| --- | --- |
| 원두명 | 필수 |
| 로스터리명 | 필수 |
| 가격 | 필수 |
| 용량 | 필수 |
| 100g당 가격 | 필수 |
| 원산지 | 필수 |
| 로스팅 정도 | 필수 |
| 컵노트 | 필수 |
| 추천 추출 방식 | 필수 |
| 산미 | 필수 |
| 단맛 | 필수 |
| 쓴맛 | 필수 |
| 바디감 | 필수 |
| 향미 강도 | 선택 |
| 밸런스 | 선택 |
| 판매 상태 | 필수 |
| 판매처 이동 | 필수 |

### 12.5 동작 규칙

- 같은 원두는 비교함에 중복 추가할 수 없다.
- 4개가 담긴 상태에서 추가를 시도하면 최대 개수 안내를 표시한다.
- 사용자는 비교함에서 원두를 제거할 수 있다.
- 비교함을 비울 수 있다.
- 비교 화면에서 각 원두 상세 화면으로 이동할 수 있다.
- 비교함은 브라우저 새로고침 후에도 유지되는 것을 목표로 한다.

### 12.6 수용 기준

- 사용자는 검색 결과에서 원두를 비교함에 추가할 수 있다.
- 사용자는 최대 4개 원두의 가격, 맛 점수, 컵노트를 한 화면에서 비교할 수 있다.
- 사용자는 비교함에서 원두를 제거할 수 있다.
- 같은 원두를 중복 추가할 수 없다.

## 13. FS-008 취향 기반 카테고리

### 13.1 목적

검색어를 잘 모르는 사용자도 취향 또는 상황에 맞는 원두 목록으로 바로 진입할 수 있게 한다.

### 13.2 MVP 카테고리

| 카테고리 | 기본 조건 |
| --- | --- |
| 신맛 적은 원두 | acidity 1-2 |
| 산미 있는 원두 | acidity 4-5 |
| 고소한 원두 | nutty 계열 컵노트 또는 고소함 태그 |
| 묵직한 원두 | body 4-5 |
| 라떼용 원두 | brew_method에 latte 또는 espresso 포함 |
| 핸드드립용 원두 | brew_method에 hand_drip 포함 |
| 2만원 이하 원두 | price <= 20000 |
| 가성비 원두 | price_per_100g이 하위 30% |
| 디카페인 원두 | is_decaf = true |

### 13.3 화면 요소

- 카테고리명
- 카테고리 설명
- 기본 적용 조건
- 원두 목록
- 정렬 옵션
- 추가 필터

### 13.4 동작 규칙

- 카테고리 화면은 검색 결과 화면과 동일한 원두 카드 형식을 사용한다.
- 카테고리 기본 조건은 사용자가 해제할 수 있다.
- 카테고리 안에서도 추가 필터와 정렬을 적용할 수 있다.

### 13.5 수용 기준

- 사용자는 "신맛 적은 원두" 카테고리에서 산미 낮은 원두 목록을 볼 수 있다.
- 사용자는 카테고리 목록에서도 가격순 정렬을 사용할 수 있다.
- 사용자는 카테고리 기본 조건 외 추가 필터를 적용할 수 있다.

## 14. FS-009 랭킹

### 14.1 목적

사용자가 가격, 취향, 사용 상황별로 상위 원두를 빠르게 확인할 수 있게 한다.

### 14.2 MVP 랭킹 유형

| 랭킹 | 기준 |
| --- | --- |
| 100g당 가격 낮은 원두 | price_per_100g 오름차순 |
| 2만원 이하 추천 원두 | price <= 20000 중 추천순 |
| 산미 강한 원두 | acidity 내림차순 후 추천순 |
| 신맛 적은 원두 | acidity 오름차순 후 추천순 |
| 라떼 추천 원두 | latte 또는 espresso 적합 원두 중 body 내림차순 |
| 핸드드립 추천 원두 | hand_drip 적합 원두 중 추천순 |
| 입문자 추천 원두 | acidity 1-3, bitterness 1-3, price <= 25000 조건 중 추천순 |

### 14.3 동작 규칙

- 각 랭킹은 최대 50개까지 표시한다.
- 기본적으로 판매 중인 원두만 포함한다.
- 랭킹 산정 기준을 화면에 짧게 표시한다.
- 랭킹 목록에서도 상세 이동, 비교함 추가, 판매처 이동이 가능하다.

### 14.4 수용 기준

- 사용자는 100g당 가격이 낮은 원두 순위를 볼 수 있다.
- 사용자는 라떼용 추천 원두 순위를 볼 수 있다.
- 사용자는 랭킹 목록에서 원두를 비교함에 추가할 수 있다.

## 15. FS-010 판매처 이동

### 15.1 목적

사용자가 마음에 드는 원두를 실제 구매 가능한 외부 판매처에서 확인할 수 있게 한다.

### 15.2 진입 위치

- 검색 결과 카드
- 원두 상세 화면
- 비교 화면
- 카테고리 목록
- 랭킹 목록

### 15.3 동작 규칙

- 판매처 링크는 새 탭으로 연다.
- 판매처 링크 클릭 시 이벤트를 기록한다.
- product_url이 없거나 유효하지 않은 원두는 사용자 화면에 노출하지 않는다.
- 품절 원두는 판매처 이동 버튼 대신 품절 상태를 표시한다.
- 제휴 링크가 있는 경우 affiliate_url을 우선 사용하고, 없으면 product_url을 사용한다.

### 15.4 수용 기준

- 사용자는 원두 카드에서 외부 판매처로 이동할 수 있다.
- 판매처 이동은 현재 검색 결과 화면 상태를 잃지 않도록 새 탭으로 열린다.
- 판매처 이동 이벤트가 기록된다.

## 16. FS-011 데이터 등록 및 검증

### 16.1 목적

운영자가 원두 데이터를 안정적으로 등록하고, 필수 데이터 누락으로 사용자 화면이 깨지는 것을 방지한다.

### 16.2 등록 방식

MVP에서는 다음 중 하나를 사용한다.

- 구조화된 JSON 파일
- 구조화된 CSV 파일
- 초기 데이터베이스 seed 파일

관리자 웹 화면은 MVP 범위에서 제외한다.

### 16.3 필수 필드

| 필드 | 조건 |
| --- | --- |
| id | 고유해야 함 |
| name | 빈 값 불가 |
| roastery_name | 빈 값 불가 |
| origin_country | 빈 값 불가 |
| roast_level | 허용된 값만 가능 |
| tasting_notes | 1개 이상 |
| acidity | 1-5 정수 |
| sweetness | 1-5 정수 |
| bitterness | 1-5 정수 |
| body | 1-5 정수 |
| recommended_brew_methods | 1개 이상 |
| price | 0보다 큰 정수 |
| weight_g | 0보다 큰 정수 |
| price_per_100g | 계산 가능해야 함 |
| product_url | URL 형식 |
| is_decaf | boolean |
| is_available | boolean |
| last_checked_at | 날짜 형식 |

### 16.4 허용 값

roast_level:

- light
- medium
- medium_dark
- dark

recommended_brew_methods:

- hand_drip
- espresso
- latte
- cold_brew
- moka_pot
- french_press

process:

- washed
- natural
- honey
- anaerobic
- semi_washed
- blend
- unknown

### 16.5 검증 규칙

- id는 중복될 수 없다.
- price_per_100g은 price와 weight_g 기준으로 재계산할 수 있어야 한다.
- 입력된 price_per_100g과 계산값이 다르면 계산값을 우선한다.
- tasting_notes는 문자열 배열로 관리한다.
- 컵노트 표기는 검색 편의를 위해 내부 표준 태그로 매핑한다.
- last_checked_at이 90일 이상 지난 데이터는 운영자 점검 대상으로 표시한다.

### 16.6 수용 기준

- 필수 필드가 누락된 원두 데이터는 등록되지 않는다.
- 100g당 가격은 가격과 용량을 기준으로 계산된다.
- 허용되지 않은 로스팅 값이 들어오면 검증 오류가 발생한다.
- 중복 id가 있으면 검증 오류가 발생한다.

## 17. FS-012 빈 결과 및 오류 상태

### 17.1 목적

검색 결과가 없거나 데이터 문제가 있을 때 사용자가 다음 행동을 할 수 있도록 명확한 상태를 제공한다.

### 17.2 빈 결과 상태

조건:

- 검색어와 필터 조합에 맞는 원두가 없음

표시 요소:

- 검색 결과 없음 안내
- 적용된 필터 요약
- 필터 초기화 버튼
- 전체 원두 보기 버튼
- 유사 카테고리 추천

### 17.3 데이터 없음 상태

조건:

- 원두 상세 URL은 존재하지만 해당 원두가 삭제되었거나 비공개 처리됨

표시 요소:

- 원두를 찾을 수 없음 안내
- 검색 결과로 돌아가기
- 홈으로 이동

### 17.4 외부 링크 오류

조건:

- product_url이 유효하지 않거나 판매처 접근이 실패함

MVP 처리:

- 링크 클릭 전 URL 형식만 검증한다.
- 외부 사이트의 실제 응답 실패는 판매처 측 문제로 보고 별도 오류 페이지를 제공하지 않는다.

### 17.5 수용 기준

- 검색 결과가 없을 때 사용자는 필터를 초기화할 수 있다.
- 존재하지 않는 원두 상세 URL 접근 시 홈 또는 검색으로 돌아갈 수 있다.
- 오류 상태에서도 화면 레이아웃이 깨지지 않는다.

## 20. 주요 수용 테스트 시나리오

### 20.1 신맛 적은 원두 찾기

Given 사용자가 홈 화면에 있다.  
When "신맛 적은 원두" 카테고리를 선택한다.  
Then 산미 점수 1-2인 원두 목록이 표시된다.  
And 각 원두 카드에는 가격, 용량, 100g당 가격이 표시된다.

### 20.2 라떼용 원두 검색

Given 사용자가 홈 화면에 있다.  
When "라떼에 좋은 원두"를 검색한다.  
Then 추천 추출 방식에 latte 또는 espresso가 포함된 원두가 우선 표시된다.  
And 사용자는 바디감 강한 순으로 정렬할 수 있다.

### 20.3 가격 비교

Given 검색 결과 화면에 원두 목록이 표시되어 있다.  
When 사용자가 100g당 가격 낮은 순 정렬을 선택한다.  
Then price_per_100g이 낮은 원두부터 표시된다.

### 20.4 원두 비교

Given 검색 결과 화면에 원두 목록이 표시되어 있다.  
When 사용자가 원두 3개를 비교함에 추가하고 비교 화면으로 이동한다.  
Then 3개 원두의 가격, 100g당 가격, 산미, 바디감, 컵노트가 한 화면에 표시된다.

### 20.5 판매처 이동

Given 사용자가 원두 상세 화면에 있다.  
When 판매처 이동 버튼을 클릭한다.  
Then 외부 판매처가 새 탭으로 열린다.  
And outbound_clicked 이벤트가 기록된다.

### 20.6 빈 결과 처리

Given 사용자가 검색 결과 화면에 있다.  
When 너무 좁은 필터 조합을 적용해 결과가 0개가 된다.  
Then 빈 결과 안내와 필터 초기화 버튼이 표시된다.

## 21. 구현 전 확인 사항

개발 시작 전 결정해야 할 항목:

- 원두 상세 URL 구조
- 비교함 상태 저장 방식
- 이벤트 로깅 도구
