## 1. 범위

이 문서는 MVP 구현 작업을 다룬다.

포함 범위:

- 프로젝트 초기 세팅
- JSON 데이터 구조와 검증
- 검색, 필터, 정렬 service
- 앱 내부 API 또는 서버리스 API
- 홈 화면
- 검색 결과 화면
- 원두 상세 화면
- 원두 비교 화면
- 카테고리 화면
- 랭킹 화면
- 이벤트 로깅 추상화
- 테스트와 QA
- 배포 준비

제외 범위:

- 회원가입과 로그인
- 자체 결제
- 사용자 리뷰 작성
- 찜 목록 서버 저장
- 관리자 웹 화면
- 자동 크롤링
- 운영 DB 도입
- 검색엔진 도입
- 모바일 네이티브 앱

## 2. 구현 기본안

MVP는 다음 구조로 구현한다.

```text
프론트엔드 앱
  -> 앱 내부 API 또는 서버리스 API
  -> service layer
  -> JSON data store
```

기본 결정:

| 항목 | 결정 |
| --- | --- |
| 저장소 | JSON 파일 |
| 검색 | 인메모리 검색 |
| 비교함 | localStorage |
| API | `/api/v1/*` contract 유지 |
| 이벤트 | `trackEvent` 추상화 후 저장소 교체 가능하게 구현 |
| 배포 | 프론트 중심 배포, 필요 시 서버리스 API 포함 |

### 7.3 핵심 검색 매핑

| 검색어 | 조건 |
| --- | --- |
| 신맛 적은 | acidity low |
| 산미 있는 | acidity high |
| 묵직한 | body heavy |
| 라떼 | brew_method latte 또는 espresso |
| 핸드드립 | brew_method hand_drip |

- `price_per_100g_asc` 정렬이 정확히 동작한다.
- 필터 그룹 간 AND, 그룹 내부 OR 규칙이 동작한다.
- 검색 결과는 BeanCard 형태로 변환된다.

### 8.3 오류 처리 기준

- 잘못된 query는 `400 INVALID_QUERY`
- 존재하지 않는 원두는 `404 NOT_FOUND`
- 존재하지 않는 카테고리 또는 랭킹은 `404 NOT_FOUND`
- 내부 오류는 `500 INTERNAL_ERROR`

### 8.4 완료 기준

- API 응답은 `data`와 `meta.request_id`를 포함한다.
- 목록 API는 pagination을 포함한다.
- 오류 응답은 공통 오류 형식을 따른다.
- API 테스트의 핵심 케이스가 통과한다.

- 키보드로 검색, 필터, 정렬, 비교함 조작이 가능하다.
- 오류 상태에서 홈 또는 검색으로 돌아갈 수 있다.

## 16. 구현 작업 상세

### 16.1 데이터와 검색 우선 개발

초기에는 화면보다 데이터와 검색 service를 먼저 안정화한다. 이 서비스의 핵심 가치는 "취향과 가격 기준으로 원두를 찾는 경험"이기 때문에, 검색 결과가 신뢰할 수 있어야 화면 구현도 의미가 있다.

완료 산출물:

- 샘플 데이터
- 데이터 검증
- 검색 service
- 필터 service
- 정렬 service
- 추천 점수 계산

### 16.2 화면은 검색 결과부터 개발

홈보다 검색 결과 화면을 먼저 구현해도 된다. 검색 결과 화면이 원두 카드, 필터, 정렬, 비교함 추가의 중심이기 때문이다.

권장 순서:

1. 원두 카드
2. 검색 결과 목록
3. 필터 패널
4. 정렬
5. 더 보기
6. 빈 결과

### 16.3 상세와 비교는 같은 데이터 표현을 공유

상세와 비교 화면은 맛 점수, 가격, 컵노트 표시 컴포넌트를 공유한다. 컴포넌트 중복을 줄이기 위해 다음 컴포넌트를 먼저 만든다.

- 가격 표시
- 맛 점수 표시
- 컵노트 태그
- 추출 방식 태그
- 판매 상태 배지

## 17. 완료 정의

### 17.1 작업 단위 완료 정의

각 `IMP-*` 작업은 다음을 만족해야 완료로 본다.

- 명세에 맞는 동작이 구현됨
- 관련 타입이 정의됨
- 오류 상태가 고려됨
- 필요한 테스트가 작성되거나 수동 검증 기록이 있음
- 기존 핵심 흐름을 깨지 않음

### 17.2 마일스톤 완료 정의

각 마일스톤은 다음을 만족해야 완료로 본다.

- 해당 마일스톤의 P0 작업 완료
- 관련 테스트 통과
- 남은 P1/P2 작업이 명확히 기록됨
- 다음 마일스톤 진행을 막는 blocker 없음

### 17.3 MVP 완료 정의

MVP는 다음 조건을 만족해야 완료로 본다.

- 홈, 검색 결과, 상세, 비교 화면이 동작한다.
- 검색, 필터, 정렬이 동작한다.
- 원두 가격과 100g당 가격이 정확히 표시된다.
- 카테고리와 랭킹으로 원두 목록을 볼 수 있다.
- 판매처 이동이 가능하다.
- 데이터 검증이 통과한다.
- 핵심 E2E 테스트가 통과한다.
- staging QA에서 출시 차단 이슈가 없다.
