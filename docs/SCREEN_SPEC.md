# 커피 원두 검색 엔진 화면 명세서

문서 버전: v0.1  
작성일: 2026-05-08  
제품명: Bean Finder, 가칭  
연관 문서: [PRD.md](./PRD.md), [FUNCTIONAL_SPEC.md](./FUNCTIONAL_SPEC.md)  
문서 목적: Bean Finder MVP의 화면 구조, UI 요소, 상태, 사용자 액션, 화면 간 이동 규칙을 정의한다.

## 1. 범위

이 문서는 MVP에 포함되는 사용자 화면을 정의한다.

포함 화면:

- 홈
- 검색 결과
- 원두 상세
- 원두 비교
- 카테고리
- 랭킹
- 빈 결과
- 데이터 없음 및 오류 상태

제외 화면:

- 로그인
- 회원가입
- 찜 목록
- 사용자 리뷰 작성
- 관리자 웹 화면
- 결제 화면
- 취향 테스트

## 3. 공통 화면 원칙

### 3.1 정보 우선순위

원두 목록과 상세 화면에서는 다음 정보를 우선 노출한다.

1. 원두명
2. 로스터리명
3. 가격과 용량
4. 100g당 가격
5. 산미와 바디감
6. 컵노트
7. 추천 추출 방식
8. 판매처 이동

### 3.2 쉬운 표현과 전문 표현

- 목록 화면에서는 쉬운 표현을 우선 표시한다.
- 상세 화면에서는 쉬운 표현과 전문 표현을 함께 표시한다.
- 예: `신맛 적음`, `산미 2/5`, `묵직함`, `바디감 4/5`

### 3.3 가격 표현

- 판매 가격 표기: `18,000원`
- 용량 표기: `200g`
- 100g당 가격 표기: `100g당 9,000원`
- 가격과 용량이 없는 원두는 사용자 화면에 노출하지 않는다.

### 3.4 기본 상호작용

- 원두 카드는 카드 전체 클릭으로 상세 화면에 진입할 수 있다.
- 카드 내부의 `비교함 추가`, `판매처 이동` 버튼은 각각 독립 동작한다.
- 판매처 이동은 새 탭으로 열린다.
- 필터, 정렬, 검색어는 URL 쿼리로 표현되어야 한다.

### 3.5 접근성

- 모든 버튼은 키보드 포커스가 가능해야 한다.
- 색상만으로 맛 점수를 구분하지 않는다.
- 아이콘 버튼에는 접근 가능한 이름을 제공한다.
- 원두 이미지에는 원두명 또는 로스터리명을 포함한 대체 텍스트를 제공한다.
- 필터 drawer와 modal은 열렸을 때 포커스를 내부에 유지한다.

## 4. 공통 컴포넌트

### 4.1 상단 내비게이션

표시 화면:

- 모든 사용자 화면

요소:

- 서비스 로고
- 검색 입력창, 데스크톱에서는 상단 고정 검색
- 비교함 버튼
- 주요 카테고리 진입점

동작:

- 로고 클릭 시 홈으로 이동한다.
- 상단 검색 실행 시 검색 결과 화면으로 이동한다.
- 비교함 버튼 클릭 시 비교 화면으로 이동한다.
- 비교함에 담긴 원두 수를 배지로 표시한다.

모바일:

- 검색 입력창은 홈에서는 크게 표시하고, 다른 화면에서는 상단 compact 형태로 표시한다.
- 카테고리 진입점은 메뉴 버튼 안에 넣을 수 있다.

### 4.2 검색 입력창

요소:

- 텍스트 입력 필드
- 검색 실행 버튼
- placeholder 문구

placeholder 예시:

- `신맛 적은 원두 검색`
- `라떼에 좋은 원두 검색`
- `에티오피아 베리향 원두 검색`

동작:

- Enter 입력 또는 검색 버튼 클릭으로 검색을 실행한다.
- 빈 검색어 제출 시 전체 원두 목록으로 이동한다.
- 기존 검색어가 있는 화면에서는 입력창에 현재 검색어를 표시한다.

### 4.3 필터 패널

표시 화면:

- 검색 결과
- 카테고리

데스크톱:

- 좌측 사이드바에 고정 표시한다.

모바일:

- `필터` 버튼 클릭 시 하단 drawer 또는 전체 화면 패널로 표시한다.

필터 그룹:

- 가격대
- 100g당 가격
- 산미
- 바디감
- 로스팅
- 원산지
- 컵노트
- 추출 방식
- 디카페인
- 판매 상태

동작:

- 필터 변경 시 결과를 갱신한다.
- `필터 초기화`는 검색어를 유지하고 필터만 제거한다.
- `전체 초기화`는 검색어, 필터, 정렬을 모두 기본값으로 되돌린다.

### 4.4 정렬 선택

표시 화면:

- 검색 결과
- 카테고리
- 랭킹 일부

옵션:

- 추천순
- 가격 낮은 순
- 100g당 가격 낮은 순
- 평점 높은 순
- 리뷰 많은 순
- 산미 강한 순
- 바디감 강한 순
- 최신 등록순

동작:

- 선택 즉시 목록을 정렬한다.
- 검색어와 필터 조건은 유지한다.

### 4.5 원두 카드

표시 화면:

- 검색 결과
- 카테고리
- 랭킹
- 원두 상세의 유사 원두 영역

필수 요소:

- 대표 이미지
- 원두명
- 로스터리명
- 가격
- 용량
- 100g당 가격
- 원산지
- 로스팅 정도
- 산미 요약
- 바디감 요약
- 컵노트 최대 3개
- 추천 추출 방식 최대 2개
- 비교함 추가 버튼
- 판매처 이동 버튼

상태:

| 상태 | 표시 |
| --- | --- |
| 기본 | 비교함 추가, 판매처 이동 가능 |
| 비교함에 있음 | 비교함 추가 버튼을 선택된 상태로 표시 |
| 품절 | 품절 배지 표시, 판매처 이동 비활성화 |
| 이미지 없음 | 기본 이미지 표시 |

### 4.6 맛 점수 컴포넌트

표시 화면:

- 원두 카드
- 원두 상세
- 원두 비교

표시 방식:

- 목록에서는 산미와 바디감 중심으로 compact 표시
- 상세와 비교에서는 산미, 단맛, 쓴맛, 바디감, 향미, 밸런스 표시

필수 조건:

- 점수 숫자와 라벨을 함께 제공한다.
- 색상만으로 높고 낮음을 표현하지 않는다.

### 4.7 비교함 미니 바

표시 화면:

- 검색 결과
- 카테고리
- 랭킹
- 원두 상세

요소:

- 담긴 원두 수
- 비교하기 버튼
- 전체 비우기 버튼

동작:

- 비교함에 1개 이상 담기면 화면 하단 또는 상단에 표시한다.
- 비교하기 버튼 클릭 시 비교 화면으로 이동한다.
- 최대 4개까지만 담을 수 있다.

## 5. 반응형 기준

| 구간 | 기준 | 레이아웃 |
| --- | --- | --- |
| 모바일 | 0-767px | 단일 컬럼, 필터 drawer, 카드 1열 |
| 태블릿 | 768-1023px | 카드 2열, 필터 drawer 또는 접이식 사이드바 |
| 데스크톱 | 1024px 이상 | 필터 좌측 사이드바, 카드 3열 이상 |

공통:

- 버튼 텍스트는 줄바꿈되더라도 영역 밖으로 넘치면 안 된다.
- 원두명은 목록 카드에서 최대 2줄까지 표시하고 초과 시 말줄임한다.
- 비교 화면은 모바일에서 가로 스크롤 또는 카드형 세로 비교를 허용한다.

## 6. SC-001 홈

### 6.1 목적

사용자가 검색어 입력, 취향 카테고리 선택, 랭킹 진입을 통해 원두 탐색을 시작한다.

### 6.2 URL

```text
/
```

### 6.3 관련 기능

- FS-001 홈 탐색
- FS-002 키워드 검색
- FS-008 취향 기반 카테고리
- FS-009 랭킹

### 6.4 주요 요소

- 상단 내비게이션
- 메인 검색 입력창
- 추천 검색어
- 취향 카테고리 목록
- 가격대별 랭킹 진입점
- 입문자 추천 진입점

### 6.5 기본 레이아웃

```text
[상단 내비게이션]

[메인 검색 영역]
  검색 입력창
  추천 검색어

[취향 카테고리]
  신맛 적은 원두 | 고소한 원두 | 묵직한 원두 | 라떼용 원두 ...

[랭킹 진입]
  100g당 가격 낮은 원두 | 2만원 이하 추천 | 입문자 추천
```

### 6.6 액션

| 액션 | 결과 |
| --- | --- |
| 검색어 입력 후 실행 | `/search?q={query}`로 이동 |
| 빈 검색어로 검색 | `/search`로 이동 |
| 추천 검색어 클릭 | 해당 검색어로 검색 결과 이동 |
| 카테고리 클릭 | `/categories/{categoryKey}`로 이동 |
| 랭킹 클릭 | `/rankings/{rankingKey}`로 이동 |

### 6.7 상태

| 상태 | 설명 |
| --- | --- |
| 기본 | 검색창과 추천 카테고리 표시 |
| 검색어 입력 중 | 입력값 유지, Enter로 검색 가능 |
| 데이터 로딩 실패 | 핵심 탐색 링크는 유지하고 일부 추천 영역에 오류 표시 |

### 6.8 수용 기준

- 사용자는 홈에서 검색을 실행할 수 있다.
- 사용자는 홈에서 주요 취향 카테고리로 이동할 수 있다.
- 사용자는 홈에서 랭킹 화면으로 이동할 수 있다.

## 7. SC-002 검색 결과

### 7.1 목적

사용자가 검색어, 필터, 정렬을 조합해 조건에 맞는 원두를 찾고 상세 또는 비교로 이동한다.

### 7.2 URL

```text
/search?q={query}&price_range={priceRange}&acidity={acidity}&body={body}&sort={sortKey}
```

예시:

```text
/search?q=라떼&body=heavy&sort=price_per_100g_asc
```

### 7.3 관련 기능

- FS-002 키워드 검색
- FS-003 필터 검색
- FS-004 검색 결과 정렬
- FS-005 검색 결과 목록
- FS-007 원두 비교
- FS-010 판매처 이동
- FS-012 빈 결과 및 오류 상태
- FS-013 기본 이벤트 로깅

### 7.4 주요 요소

- 상단 내비게이션
- 검색 조건 요약
- 결과 수
- 필터 패널
- 정렬 선택
- 적용 필터 chip
- 원두 카드 목록
- 더 보기 버튼
- 비교함 미니 바

### 7.5 기본 레이아웃

데스크톱:

```text
[상단 내비게이션]

[검색 조건 요약] [정렬]

[필터 사이드바] [원두 카드 그리드]
               [더 보기]

[비교함 미니 바]
```

모바일:

```text
[상단 내비게이션]
[검색 조건 요약]
[필터 버튼] [정렬]
[원두 카드 1열]
[더 보기]
[비교함 미니 바]
```

### 7.6 액션

| 액션 | 결과 |
| --- | --- |
| 검색어 변경 후 실행 | 쿼리 갱신 후 결과 재조회 |
| 필터 선택 | URL 쿼리 갱신, 결과 재조회 |
| 필터 초기화 | 검색어 유지, 필터 제거 |
| 전체 초기화 | `/search`로 이동 |
| 정렬 변경 | sort 쿼리 갱신, 결과 재정렬 |
| 원두 카드 클릭 | `/beans/{beanId}`로 이동 |
| 비교함 추가 | 비교함에 원두 추가 |
| 판매처 이동 | 새 탭으로 외부 링크 열기 |
| 더 보기 클릭 | 다음 20개 결과 추가 표시 |

### 7.7 상태

| 상태 | 설명 |
| --- | --- |
| 로딩 | 목록 영역에 skeleton 표시 |
| 결과 있음 | 원두 카드 목록 표시 |
| 결과 없음 | SC-007 빈 결과 상태 표시 |
| 필터 오류 | 잘못된 가격 범위 등 입력 오류 표시 |
| 데이터 오류 | 목록 대신 오류 안내와 재시도 표시 |

### 7.8 원두 카드 표시 규칙

- 카드에는 대표 컵노트를 최대 3개 표시한다.
- 추천 추출 방식은 최대 2개 표시한다.
- 산미와 바디감은 목록에서 반드시 표시한다.
- 이미지가 없으면 기본 이미지를 사용한다.
- 품절 원두는 `품절` 배지를 표시하고 판매처 이동을 비활성화한다.

### 7.9 수용 기준

- 사용자는 검색 결과에서 가격, 용량, 100g당 가격을 볼 수 있다.
- 사용자는 필터와 정렬을 조합할 수 있다.
- 사용자는 원두를 비교함에 추가할 수 있다.
- 사용자는 검색 결과에서 상세 화면으로 이동할 수 있다.

## 8. SC-003 원두 상세

### 8.1 목적

사용자가 특정 원두의 맛, 가격, 원산지, 로스팅, 추천 추출 방식, 유사 원두를 상세히 확인한다.

### 8.2 URL

```text
/beans/{beanId}
```

### 8.3 관련 기능

- FS-006 원두 상세 조회
- FS-007 원두 비교
- FS-010 판매처 이동
- FS-012 빈 결과 및 오류 상태
- FS-013 기본 이벤트 로깅

### 8.4 주요 요소

- 상단 내비게이션
- 뒤로 가기 또는 검색 결과로 돌아가기
- 대표 이미지
- 원두명
- 로스터리명
- 가격 정보

### 12.2 표시 위치

- 검색 결과
- 카테고리

### 12.3 표시 요소

- 결과 없음 안내
- 현재 검색어
- 적용된 필터 요약
- 필터 초기화 버튼
- 전체 원두 보기 버튼
- 추천 카테고리 링크

### 12.4 액션

| 액션 | 결과 |
| --- | --- |
| 필터 초기화 | 검색어 유지, 필터 제거 |

| 추천 카테고리 클릭 | 해당 카테고리로 이동 |

### 12.5 수용 기준

- 사용자는 결과가 없는 이유를 현재 조건 기준으로 이해할 수 있다.
- 사용자는 한 번의 액션으로 필터를 초기화할 수 있다.
- 사용자는 홈으로 돌아가지 않고도 다른 탐색을 이어갈 수 있다.

## 13. SC-008 데이터 없음 및 오류 상태

### 13.1 목적

존재하지 않는 URL, 삭제된 원두, 데이터 로딩 실패 상황에서 사용자가 탐색을 계속할 수 있게 한다.

### 13.2 원두 없음

조건:

- `/beans/{beanId}`에 접근했지만 원두가 존재하지 않음

표시 요소:

- 원두를 찾을 수 없음 안내
- 홈으로 이동 버튼
- 전체 원두 보기 버튼

### 13.3 카테고리 또는 랭킹 없음

조건:

- 존재하지 않는 categoryKey 또는 rankingKey로 접근

표시 요소:

- 페이지를 찾을 수 없음 안내
- 홈으로 이동 버튼
- 주요 카테고리 링크

### 13.4 데이터 로딩 실패

조건:

- 서버 또는 데이터 소스 문제로 화면 데이터를 불러오지 못함

표시 요소:

- 오류 안내
- 재시도 버튼
- 홈으로 이동 버튼

### 13.5 수용 기준

- 존재하지 않는 원두 URL에서도 화면이 깨지지 않는다.
- 사용자는 오류 상태에서 홈 또는 검색으로 이동할 수 있다.
- 데이터 로딩 실패 시 재시도할 수 있다.
